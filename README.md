# Cloud Security Implementation: IAM, Secure Storage, and Data Encryption

## Task Overview
This task was completed as part of the **CODTECH Cloud Computing Internship**.  
The objective was to implement cloud security best practices by configuring **Identity and Access Management (IAM)**, securing cloud storage, and enabling **data encryption** to protect resources and sensitive information.

## Technologies and Services Used
### Amazon Web Services (AWS)
- AWS IAM
- Amazon S3
- AWS Key Management Service (KMS)

### Google Cloud Platform (GCP)
- Cloud Storage
- Cloud IAM
- Cloud Key Management Service (KMS)

## Features Implemented
- Configured **IAM users, roles, and policies** to enforce secure access control.
- Applied **least-privilege permissions** for cloud resources.
- Enabled **server-side encryption** for secure cloud storage.
- Configured **data protection** using encryption keys.
- Verified secure access and protected sensitive data across cloud services.

## Project Architecture
1. Created IAM users and assigned roles/policies.
2. Configured secure cloud storage buckets.
3. Enabled encryption for stored data.
4. Managed encryption keys using cloud KMS services.
5. Tested access permissions and security controls.

## Steps Performed
1. Created IAM users and custom policies.
2. Assigned secure permissions to cloud resources.
3. Configured secure storage in AWS S3 / Cloud Storage.
4. Enabled encryption using KMS-managed keys.
5. Tested access restrictions and encryption behavior.
6. Verified secure cloud resource management.
